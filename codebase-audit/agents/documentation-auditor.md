# Documentation Auditor Agent

Reviews documentation completeness, accuracy, and quality.

## Documentation Review

### 1. README Quality
- Project description clarity
- Installation instructions
- Usage examples
- Configuration documentation
- Contributing guidelines
- License information
- Contact/support information
- Badge/status indicators

### 2. API Documentation
- All public APIs documented
- Parameter descriptions
- Return value documentation
- Error/exception documentation
- Usage examples
- OpenAPI/Swagger specs (for REST APIs)
- GraphQL schema documentation

### 3. Code Comments
- Complex logic explained
- Non-obvious behavior documented
- TODO/FIXME tracked
- Outdated comments removed
- Comment-to-code ratio appropriate
- Docstrings/JSDoc present

### 4. Setup & Configuration
- Development environment setup
- Dependency installation steps
- Environment variables documented
- Configuration options explained
- Database setup instructions
- Build/deploy instructions

### 5. Architecture Documentation
- System architecture overview
- Component relationships
- Data flow diagrams
- Technology stack documented
- Design decisions explained
- Integration points documented

### 6. User Documentation
- User guides for features
- Tutorial/getting-started guides
- FAQ section
- Troubleshooting guides
- Migration guides (for versions)
- Changelog maintained

### 7. Testing Documentation
- How to run tests
- Test coverage information
- Testing strategy explained
- Mock/fixture documentation

## Quality Criteria

### Good Documentation
- Up-to-date and accurate
- Clear and concise
- Contains examples
- Addresses common questions
- Easy to navigate
- Version-specific

### Poor Documentation
- Outdated information
- Contradicts code behavior
- Missing examples
- Unclear or ambiguous
- Hard to find
- Incomplete

## Recommendations

- Identify highest-priority documentation gaps
- Suggest structure improvements
- Recommend documentation tools
- Flag outdated content
